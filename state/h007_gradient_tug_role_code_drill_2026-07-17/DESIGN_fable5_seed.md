## 2. The one-variable lever — the SECOND objective's content

One shared trunk (d=384 L=4, the H_003/H_004 envelope), TWO heads: head-A = forward next-byte CE
(always, all arms); head-G = a second per-position byte-CE whose TARGET is the arm variable. Corpus,
model, params, steps, init, seeds byte-identical across arms; head-G's architecture identical; ONLY the
target sequence it is trained to predict differs. Heads are dropped at eval; all panel scoring is
head-A forced-choice free-slot d_acc (metric conventions carried; f1/f2 are PANEL names).

The tug objective, concrete at byte level: the drill grammar uses FIXED-BYTE-LENGTH noun-class pools, so
the role-swapped rendering swap(x) (subject NP ↔ object NP fillers exchanged, markers kept in place) is
positionally byte-aligned with x by construction. head-G at position t predicts swap(x)[t+1]. At swapped-NP
positions head-A must predict the noun that IS there while head-G predicts the one that ISN'T — the shared
state at the SAME position must encode both fillers AND which role each occupies. Dense, per-position,
unsolvable by collocation memorization (`mech-5.weakness`'s own requirement). On non-drill corpus text
(no two-role frame), head-G's target = the forward target (all arms identical there) — the arm delta is
confined to the drill-register portion, which is thousands of full sentences × all steps, NOT a 192-item
identification problem.

| arm | head-G target | isolates |
|---|---|---|
| **A-tug** | swap(x), role-swapped aligned rendering | the mechanism (irreconcilable second objective) |
| **C-dup** | x itself (duplicate forward CE) | **the control** — same params/compute/gradient volume, zero conflict, zero new information |
| **C-shuf** | swap(x′), x′ a DIFFERENT sentence of the same template class (length-aligned) | placebo — "any hard second task regularizes" vs role structure specifically |
| **C-scaf** | no second loss (head-G present, λ=0) | floor; also mech-5's own L2 "ablation retrain" realized as an arm |
| **C-perm** | A-tug trained on permuted drill gold | harness |

**Supervision-leak BAN (the raw-hon-φ trap, named up front):** head-G's target must be a full-sequence
rendering, NEVER a classifier of the panel's gold bit or its ingredients (class(subject), marker form).
A G-objective that supervises the gold bit certifies supervision, not tension — the exact `dont`.

## 3. The panel — SWAP-XOR (role binding made visible, heuristics at 0.5)

Task: two role-marked NPs + verb, **scrambled orders counterbalanced** (SOV/OSV via case markers — Korean
licenses this); forced choice on the verb's agreement suffix. Synthetic drill-grammar rule (H_003/H_004
precedent licenses constructed rules): **gold = class(subject) ⊕ form(object-marker)**, where noun classes
are {honored, plain} (fixed-byte-length pools) and the object marker has two drilled forms (e.g. 를 vs a
synthetic variant), both bits 50/50 and mutually decorrelated. The two gold ingredients live at two
NON-ADJACENT sites, and site 1 is defined by the subject marker under scrambling — no single local window
computes gold; the trunk must find the subject by its marker, read its class, and compose it with the
distant object-marker form. That composition is the measured-dead capability of the unaided trunk
(0.49–0.62 band) and the exact thing a role-abstracted code would buy.

- Heuristics that must sit at 0.500 closed-form (F7, before any training): class-of-first-noun ·
  class-of-noun-nearest-verb · class-presence/majority · object-marker-form alone · class(subject) alone ·
  any single-site byte n-gram (the panel is counterbalanced so each ingredient alone is 0.5).
- Panels: **f2 (verdict, OOD)**: held-out (N1,N2) pairings × both orders × both marker forms × both
  classes, fully crossed, 50/50 per cell, n=192 (σ≈0.036, H_003 convention). **f1 (liveness)**: drilled
  combos, held-out conjugation/lexeme, n=64.
- **Codebook/leak audit up front (the metric trap, priced in):** recompute the free-slot set for THIS
  panel (never inherit H_004's {0,1,2,4}); GF(2)-rank audit of the {class-bits, marker-bits, gold} code
